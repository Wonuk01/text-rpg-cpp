# ⚔️ 텍스트 RPG 전투 시뮬레이터

C++17로 구현한 콘솔 텍스트 RPG 게임입니다.  
캐릭터를 생성하고 몬스터와 턴제 전투를 벌이는 텍스트 기반 RPG입니다.

---

## 실행 방법

> `rpg.exe`와 `bgm.mp3`가 **반드시 같은 폴더**에 있어야 합니다.

1. 이 저장소를 ZIP으로 다운로드 후 압축 해제
2. `rpg.exe` 실행

※ Windows 전용

---

## 빌드 방법

MinGW (g++) 필요:

**게임 실행파일:**
```bash
g++ -std=c++17 -o rpg main.cpp Entity.cpp Character.cpp Monster.cpp BattleManager.cpp Item.cpp Jobs.cpp SaveManager.cpp -lwinmm
```

**단위 테스트:**
```bash
g++ -std=c++17 -o test_run test.cpp Entity.cpp Character.cpp Jobs.cpp Item.cpp
```

## 주요 기능 사용법

### 1. 캐릭터 생성
- 게임 시작 후 `[1] 새게임` 선택
- 이름 입력 → 직업 선택 (전사 / 마법사 / 힐러)

| 직업 | HP | 공격력 | 방어력 | 특징 |
|------|-----|--------|--------|------|
| 전사 | 120 | 18 | 8 | 강공격 패턴, Lv.2 분노의 일격 |
| 마법사 | 80 | 25 | 3 | 마나 소모 고데미지, Lv.2 마력 폭발 |
| 힐러 | 100 | 14 | 5 | 매 턴 자동 회복, Lv.2 신성 치유 |

### 2. 전투 시스템
전투 중 선택 가능한 행동:
- `[1] 공격` — 기본 공격 (15% 확률로 크리티컬 1.5배)
- `[2] 아이템 사용` — 인벤토리에서 아이템 선택 후 사용
- `[3] 도망` — 50% 확률로 전투 탈출
- `[4] 스킬 사용` — Lv.2 달성 후 사용 가능 (스킬 보유 시에만 표시)

### 3. 아이템 & 인벤토리
- 스테이지 클리어 후 마을 상점에서 구매
- 구매한 아이템은 인벤토리에 보관되며 전투 중 `[2] 아이템 사용`으로 꺼내 사용

| 아이템 | 가격 | 효과 |
|--------|------|------|
| HP 포션 | 30G | HP +50 회복 |
| 공격 포션 | 50G | ATK +8 강화 |
| 방어 강화 | 50G | DEF +3 즉시 적용 |

### 4. 저장 / 불러오기
- 마을 메뉴에서 `[3] 저장` 선택 → `savefile.txt`에 저장
- 메인 메뉴에서 `[2] 불러오기` 선택 → 저장된 스테이지부터 재개

### 5. 설정 (배경음악)
- 메인 메뉴 `[3] 설정` → `[1]` 볼륨 증가 / `[2]` 볼륨 감소 / `[3]` 음소거

## 스테이지 구성

| 스테이지 | 몬스터 | 특징 |
|----------|--------|------|
| 1 | 고블린 | 70% 확률 연속 공격 |
| 2 | 늑대 | 울부짖음 후 다음 턴 1.8배 강화 공격 |
| 3 | 악어 | 3턴마다 데스롤 (1.8배 데미지) |
| 4 | 드래곤 | 3턴마다 불꽃 브레스 (2배 데미지) |

## 파일 구조

```
rpg/
├── rpg.exe             실행파일 (Windows)
├── bgm.mp3             배경음악 (필수, exe와 같은 폴더)
├── test_run.exe        단위 테스트 실행파일
├── main.cpp            메인 루프 / 메뉴 / 오디오
├── Entity.cpp/h        공통 추상 클래스 (HP, 공격, 방어)
├── Character.cpp/h     플레이어 캐릭터 추상 클래스
├── Jobs.cpp/h          Warrior / Mage / Healer 구현
├── Monster.cpp/h       Goblin / Wolf / Crocodile / Dragon 구현
├── BattleManager.cpp/h 턴제 전투 시스템
├── SaveManager.cpp/h   저장·불러오기 (파일 I/O)
├── Item.cpp/h          아이템 클래스
├── test.cpp            단위 테스트 (main과 별도 실행)
└── report.docx         프로젝트 보고서
```
